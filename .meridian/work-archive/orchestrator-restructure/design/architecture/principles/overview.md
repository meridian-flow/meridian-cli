# A05: Principles — Subsystem Overview

## Purpose

This subsystem describes how `dev-principles` is applied across the orchestrator topology: as universal shared guidance loaded by every agent whose work is shaped by structural, refactoring, abstraction, or correctness concerns, never as a binary pass/fail gate at any altitude. This overview is a strict TOC; substantive content lives in the single leaf.
